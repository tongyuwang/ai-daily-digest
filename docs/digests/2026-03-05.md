What makes it stand out from the crowded agent framework space: it's designed at the *business process* level, not the individual task level. ### 2. Boris Cherny (Creator of Claude Code): What Happens When We Stop Writing Code
**[@GergelyOrosz (The Pragmatic Engineer)](https://x.com/GergelyOrosz/status/2029295486574747872)** · 👁 282K impressions · 🔖 2,700 bookmarks

A wide-ranging interview with Boris Cherny — the creator of Claude Code — from The Pragmatic Engineer. The deepest look yet at how Claude Code was built, how the team actually uses it, and what it means for software engineering culture.

Three standout insights from the transcript:

1. **Boris automated himself out of code review before AI existed.** Every recurring review comment went into a spreadsheet; once a pattern hit 3–4 occurrences, he wrote a lint rule. A preview of the mindset that built Claude Code.

2. **PRDs are dead on the Claude Code team — prototypes replaced them.** Instead of writing Product Requirements Documents, the team builds hundreds of working prototypes before shipping. *"There's just no way we could have shipped this if we started with static mocks and Figma or if we started with a PRD."*

3. **2026 is the year of the generalist — and maybe the year of people with ADHD.** Agentic work has shifted Boris from deep single-threaded focus to managing parallel agents with rapid context switching. *"It's not so much about deep work, it's about how good I am at context switching and jumping across multiple different contexts very quickly."*

Also covered: Claude Code's architecture, permissions/sandboxing, agent swarms, Claude Cowork, and what skills still matter for engineers.

### 4. OpenAI Symphony: Agent Orchestration in Elixir (96.1%)
**[Tweet](https://x.com/user/status/2029327139196289358)** · 👁 32.9K impressions · 🔖 399 bookmarks

OpenAI open-sourced their internal agent orchestration system — and it's almost entirely written in Elixir. The official rationale:
- Excellent supervision of long-running processes (BEAM/OTP's killer feature)
- Hot code reloading without stopping active subagents (critical for dev iteration)
- Active ecosystem of libraries

Symphony "turns project work into isolated, autonomous implementation runs, allowing teams to manage work instead of supervising coding agents." The Elixir community is understandably delighted. For the broader AI ecosystem, this is a strong signal that production agent infrastructure may trend toward functional/actor-model languages where fault tolerance is a primitive, not an afterthought.

### 5. LangWatch: Open-Source LLM Eval & Agent Testing Platform
**[Tweet](https://x.com/user/status/2029173031621509234)** · 👁 62K impressions · 🔖 1,115 bookmarks

LangWatch positions itself as the missing observability layer for AI agents: trace → dataset → evaluate → optimize → re-test, with zero glue code. The full feature set:
- End-to-end agent simulations (tools, state, user simulator, judge)
- Closed eval loop with optimization studio
- GitHub integration — prompt versions live in Git, linked to traces
- OpenTelemetry-native, framework-agnostic (LangChain, LangGraph, CrewAI, Vercel AI SDK, Mastra, Google ADK)
- Model-agnostic (OpenAI, Anthropic, Azure, AWS, Groq, Ollama)
- Full MCP support for Claude Desktop
- ISO 27001 certified, 100% open source, one `docker compose` command to self-host

The framing in the post is a bit hyped, but the underlying tool checks out. Most teams shipping AI agents have zero regression testing — LangWatch makes that inexcusable.
